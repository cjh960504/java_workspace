javac.exe:컴파일
컴파일이란? 원본소스를 기계어로 변환하는 과정
		이 과정에서 문법, 오류를 검사 (번거롭고 느리지만)
		최초에 한번만 성공해놓으면, 그떄부터 기계어를 이용해서  실행하므로 속도가 빠름
인터프리터 방식 : 자바스크립트, 파이썬, R..
			원본소르를 기계어로 변환하는 과정을 개발자가 하지 않는다.. 내부적으로 처리
			개발자는 편하지만, 매번 수행할때마다 문법, 해석 등을 진행하므로 속도는 느리다...
			R을 이용하여 분석할 떄, 한줄씩 실행하면서 결과를 봐야하므로, 이떄 인터프리터가 편하다!

자바로 개발할 수 있는 분야는? 개발플랫폼
- javaSE(Standard Edition) : 자바의 표준 개발 플랫폼 
= 응용프로그램 개발 가능, desktop, notebook 등 컴퓨터에 직접 설치하여 실행되는 설치형 프로그램
    장점 ) pc 성능이 지원하는 한 모든지 할 수 있음
- javaEE(Enterprise Edition) : 기업용 개발 플랫폼 (웹용)
= 웹어플리케이션 개발 시 필요한 기술, Jsp/servlet...
-javaME(Micro Edition) : 하드웨어에 포함되는 작은 규모의 프로그램 (임베디드 프로그램)
				     세탁기, 냉장고, 자동차, 스마트폰...
				     javaME보다는 안드로이드

Write Once Run Anywhere!
- 한번 작성하고 어디서든지 작동! => .class (= 바이트코드)를 OS가 실행하는 것이 아니라 JVM이 해석하여 실행!

Java Virtual Machine	: .class 파일들을 실행하기 위한 장치?	
JDK : Java Development Kit = 자바개발도구, bin에 실행파일 
JRE : Java Runtime Environment = 자바 실행 환경 (JVM을 가지고 있는 포함?)
개발자는 둘다 설치, 실행만 하는 사람들은 JRE만
